# 要件定義書：N体重力シミュレーション Web アプリ

**プロジェクト名：** Solar Gravity Simulator  
**技術スタック：** Rust + WebAssembly（WASM）+ TypeScript / HTML / CSS  
**開発期間：** 3日間（2人）  
**対応環境：** ブラウザ（PC・スマホ・タブレット対応）  
**最終更新：** 2026-04-29

---

## 1. 概要

太陽系を舞台にしたN体重力シミュレーション。惑星が互いの重力で引き合う様子をリアルタイムで可視化し、地球からロケット・人工衛星を打ち上げてその軌道をシミュレートできるインタラクティブWebアプリ。地球から月を目指すミッションを中心に、太陽・地球・月などの引力バランスを調整しながら、うまく制御しないと太陽側へ引っ張られてしまうような「触って試せる」遊びを提供する。演出要素としてUFOも登場させ、物理計算の正確さと見た目の面白さをバランスよく両立する。

---

## 2. 機能要件

### 2-1. 天体シミュレーション

| 項目 | 内容 |
|---|---|
| 登場天体 | 太陽、水星、金星、地球、月、火星（計6天体・固定配置） |
| 物理モデル | N体問題（全天体ペア間の万有引力を毎ステップ計算） |
| 計算精度 | Rust側でサブステップ積分（Velocity Verlet法） |
| 初期配置 | 太陽系の軌道半径比に基づく固定配置（カスタマイズ不要） |
| 重力パラメータ調整 | 太陽・地球・月など主要天体の重力の強さをUIで調整できる |
| 月到達ミッション | 地球から月へ向かうロケットを発射し、月付近でうまく速度調整できないと太陽や他天体の重力に引っ張られる挙動を楽しめる |
| 軌道トレイル | 各天体の軌跡を一定長さで描画 |
| 軌道ガイド | 各惑星の公転軌道を薄い円で表示（トグル可） |

### 2-2. ロケット・衛星シミュレーション

操作方法は**パラメータ入力とマウス操作の両方**をサポートする。

#### パラメータ入力モード
- 発射角度：スライダー + 数値入力（0〜360°）
- 初速：スライダー + 数値入力（ゲーム内 km/s スケール）
- 「発射」ボタンで打ち上げ

#### マウス操作モード
- 地球をクリックし、ドラッグして方向と速度を指定
- ドラッグ中に矢印で方向・長さ（＝初速）を可視化
- 離したときに発射

#### 共通仕様

| 項目 | 内容 |
|---|---|
| 発射地点 | ロケットは地球上または地球近傍から発射する |
| 同時発射数 | 複数ロケットを同時に飛ばせる（上限5機程度） |
| 衛星モード | 円軌道に乗せるための初速を自動計算するモード |
| 月到達チャレンジ | 地球から月へ向かうことを基本ミッションとして扱う |
| 軌道トレイル | ロケットの飛行経路を色付きで表示 |
| 到達判定 | 惑星・月への到達を検出してメッセージ表示 |
| 失敗判定 | 太陽へ落下・太陽方向へ大きく引っ張られる・画面外へ飛ぶなどを分かりやすく表示 |
| 予測軌道 | 発射前に数秒先の軌道を点線でプレビュー表示（Nice to have） |


### 2-3. UFO・演出要素

物理シミュレーション本体とは別に、見た目の楽しさとインタラクティブ性を高めるため、UFOを登場させる。

| 項目 | 内容 |
|---|---|
| UFO表示 | 宇宙空間にUFOを表示する |
| UFOの役割 | 障害物・観察者・お楽しみキャラクターのいずれかとして扱う（MVPでは見た目優先でOK） |
| UFOとのインタラクション | ロケットが近づいたときにリアクション表示やメッセージを出す（Nice to have） |
| UFOの物理影響 | 基本は重力計算に含めない。余裕があれば弱い引力や反発などの遊び要素を追加する |

### 2-4. テレメトリー表示

選択中のロケット・衛星について以下をリアルタイム表示：

- 現在速度（km/s スケール）
- 地球からの高度
- 受けている重力加速度
- 最も近い天体名と距離
- 月到達・太陽落下・画面外などのミッション状態

### 2-5. UI・操作

| 項目 | 内容 |
|---|---|
| タイムスケール | スライダーで時間の進み速度を調整（一時停止・早送り） |
| カメラ | ピンチ/スクロールでズーム、ドラッグでパン |
| ズームUI | 太陽圏・地球・月・ロケットを見やすく切り替えられるズームイン/アウト操作を用意する |
| プリセットズーム | 「太陽系全体」「地球周辺」「月周辺」「ロケット追従」などの表示切り替えを用意する（MVPでは一部のみでOK） |
| 天体フォーカス | 天体をクリックするとその天体を追従カメラ |
| 重力調整UI | 太陽・地球・月など主要天体の重力倍率をスライダーで調整できる |
| リセット | 初期配置に戻すボタン。重力パラメータも初期値に戻せる |
| 軌道表示トグル | 軌道ガイド・トレイルの表示ON/OFF |
| レスポンシブ | スマホ・タブレット対応（タッチ操作） |

## 3. 非機能要件

| 項目 | 目標 |
|---|---|
| フレームレート | 60fps（PC）、30fps（スマホ） |
| 物理計算 | Rust/WASM側で実行、JSはUIと描画のみ |
| 描画 | Canvas 2D API（MVP）、余裕があればWebGL化 |
| ビルド | wasm-pack でビルド、Vite でフロント配信 |
| デプロイ | GitHub Pages 等の静的ホスティング |

## 4. 技術アーキテクチャ

```
┌──────────────────────────────────────┐
│           ブラウザ (TypeScript)       │
│  ・Canvas描画（天体・UFO・トレイル・炎）   │
│  ・UI操作（スライダー・ボタン）       │
│  ・マウス/タッチ入力ハンドリング      │
│  ・ズーム・パン・フォーカス表示制御   │
│  ・テレメトリーパネル表示            │
└──────────────┬───────────────────────┘
               │ 毎フレーム呼び出し
               ▼
┌──────────────────────────────────────┐
│         Rust / WASM モジュール        │
│  ・N体物理計算（Velocity Verlet積分） │
│  ・ロケット軌道計算                  │
│  ・衝突・到達判定                    │
│  ・重力倍率などのパラメータ管理       │
│  ・軌道予測計算                      │
│  ・全天体・ロケットの状態管理         │
└──────────────────────────────────────┘
```

### Rust/WASM が公開する主なAPI

```rust
// 物理ステップを進める
pub fn step(dt: f64);

// 全天体の現在状態を返す（位置・速度）
pub fn get_bodies() -> Vec<BodyState>;

// ロケットを発射する（rocket_id を返す）
pub fn launch_rocket(angle_deg: f64, speed: f64) -> u32;

// ロケットのテレメトリーを取得
pub fn get_telemetry(rocket_id: u32) -> Telemetry;

// 軌道予測（発射前プレビュー用）
pub fn predict_orbit(angle_deg: f64, speed: f64, steps: u32) -> Vec<(f64, f64)>;
```

## 7. 未決定事項（着手前に決めること）

| 項目 | 選択肢 | 推奨 |
|---|---|---|
| 描画方式 | Canvas 2D vs WebGL | **Canvas 2DでMVP、余裕があればWebGL** |
| 天体テクスチャ | 色のみ vs 画像テクスチャ | **色のみ（開発速度優先）** |
| スケール感 | 実天文単位 vs ゲームスケール | **ゲームスケール（見やすさ優先）** |
| ズーム範囲 | 太陽系全体を見せる vs 地球・月周辺を見せる | **両方を切り替え。MVPではプリセット中心** |
| 重力調整対象 | 全天体 vs 主要天体のみ | **太陽・地球・月を優先** |
| UFOの扱い | 演出のみ vs 物理影響あり | **MVPでは演出のみ** |
| ホーマン遷移 | 自動計算モードを入れるか | **時間があれば。Day3のNiceで判断** |
