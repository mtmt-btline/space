# N-Body Sim — 要件定義書

**バージョン:** v0.4（追記版）  
**プロダクト:** N-Body Sim  
**概要:** 重力多体シミュレーションをブラウザ上で直感的に体験できる Web アプリ  
**ステータス:** v1.0 開発計画前の要件整理版  
**参考実装:** https://github.com/mtmt-btline/space

---

## 1. 目的

N-Body Sim は、太陽系を題材にした重力多体シミュレーションを、ブラウザ上でインタラクティブに体験できる Web アプリである。

実物理スケールの厳密な再現よりも、以下の体験を優先する。

- ドラッグしてロケットを発射する楽しさ
- 重力によって軌道が曲がる直感的な理解
- 惑星配置や重力定数を変えたときの変化
- 「初期配置を変えるだけで別の宇宙が生まれる」感覚

ただし、遊びとして破綻しない軌道体験を提供するため、v1.0 から数値計算の安定性を重視する。

---

## 2. v1.0 の完成定義

v1.0 は、以下を満たした状態とする。

- 太陽系プリセットが起動時に表示される
- ユーザーが地球からロケットをパチンコ式に発射できる
- ドラッグ中に数秒先までの軌道予測線が滑らかに表示される
- 月・火星・地球・太陽への接触結果が判定される
- 惑星の位置をドラッグで変更でき、ドラッグ終了時に安定しやすい初期速度へ補正される
- 空きスペースのドラッグで新しい惑星を追加できる
- ズーム、パン、Follow Mode により広域・近距離の両方を観察できる
- Tweaks パネルで物理・表示・ロケット関連の主要パラメータを調整できる
- React 単独で動作する
- 将来的に Rust/WASM ソルバーへ差し替え可能な Solver I/F が定義されている

## 5. 機能要件

## 5.1 オブジェクト

### 5.1.1 種別

| 種別 | 説明 | 物理挙動 | 操作 |
|---|---|---|---|
| 恒星 | 主な重力源。v1.0 では太陽 | 原則固定または半固定 | 通常は移動不可 |
| 惑星 | 太陽などの周囲を公転 | 重力を受け、他天体へも影響する | 選択、ドラッグ、削除 |
| 衛星 | 惑星の周囲を公転 | 惑星と同様 | 選択、ドラッグ、削除 |
| ロケット | ユーザーが発射する軽量オブジェクト | 重力を受けるが、他天体へ影響しない | パチンコ式発射 |
| UFO | 演出用 | 物理対象外 | v1.0 では後回し |

v1.0 では、月は地球のまわりを公転する衛星として扱う。

---

### 5.1.2 半径

天体の半径は、物理判定用と表示用を分離する。

| 半径 | 用途 |
|---|---|
| `physicalRadius` | 衝突、着陸、近接判定 |
| `drawRadius` | 描画、クリック判定、視認性 |

要件:

- 重力計算には `drawRadius` を使わない
- クリック判定は `drawRadius` を基準にする
- 小さい天体も操作しやすい表示サイズに補正する
- 表示半径は質量差をそのまま反映せず、対数スケールなどで調整する

---

## 5.2 ロケット発射

### 5.2.1 基本仕様

v1.0 のメイン操作は、パチンコ式ドラッグとする。

- 地球または発射可能な天体からドラッグ開始する
- ドラッグ方向の反対向きに発射する
- ドラッグ距離が長いほど初速を大きくする
- 初速には上限を設ける
- 角度・初速スライダーはメイン UI に出さず、必要に応じて Tweaks 側に置く

### 5.2.2 発射位置

- デフォルトの発射元は地球
- 発射位置は以下を基準にする

```txt
発射位置 = 地球中心 + 発射方向 * physicalRadius
```

- 見た目上は `drawRadius` を考慮し、地表から発射しているように補正してよい

---

## 5.3 軌道予測線

### 5.3.1 目的

ドラッグ中に「この方向へ撃つとどう飛ぶか」を直感的に示す。

### 5.3.2 方針

- 滑らかな表示を最優先する
- 本シミュレーションと完全一致しなくてよい
- 発射予定のロケットのみを予測対象とする
- 惑星側の未来位置は簡略化してよい
  - 現在位置で固定
  - 現在速度で等速移動
  - 軽量ステップで近似更新
- `requestAnimationFrame` に同期して更新する
- 計算負荷が高い場合は、精度よりも応答性を優先する

### 5.3.3 制約

以下には上限を設ける。

- 予測秒数
- 予測ステップ数
- 予測線の点数
- 再計算頻度

---

## 5.4 接触判定

### 5.4.1 ロケット対天体

ロケットが天体の `physicalRadius` に接触した場合、相対速度で結果を判定する。

| 条件 | 結果 |
|---|---|
| 月に低速接触 | 月に着陸成功 |
| 月に高速接触 | 月面にクラッシュ |
| 火星に低速接触 | 火星に着陸成功 |
| 火星に高速接触 | 火星にクラッシュ |
| 地球に低速接触 | 地球に帰還 |
| 地球に高速接触 | 地球にクラッシュ |
| 太陽に接触 | 太陽に呑まれた |
| シミュレーション圏外へ脱出 | 宇宙の彼方へ |

### 5.4.2 着陸・クラッシュ

- 相対速度がしきい値以下なら着陸成功
- しきい値を超える場合はクラッシュ
- 着陸時は、以下のいずれかで表現する
  - 天体表面に固定
  - 速度を減衰させて吸着させる
- クラッシュ時は、以下のいずれかで表現する
  - ロケットを消す
  - 破片演出を出す
  - 速度を大幅に減衰させる
- ロケット対惑星では、弾性衝突によるバウンドをメイン挙動にしない

---

## 5.5 惑星・衛星の操作

### 5.5.1 選択

天体をクリックすると Inspector に以下を表示する。

- 名前
- 種別
- 質量
- 速度
- 座標
- 物理半径
- 表示半径
- Follow ボタン
- 削除ボタン

### 5.5.2 ドラッグ移動

- 惑星・衛星をドラッグすると位置を移動できる
- ドラッグ中は、対象天体のユーザー操作を優先する
- 必要に応じてシミュレーションを一時停止する
- ドラッグ終了時に速度ベクトルを補正する

### 5.5.3 動的円軌道補正

惑星・衛星をドラッグして離した瞬間、以下の手順で速度を補正する。

1. 対象天体に最も強い重力影響を与えている天体を探す
2. その天体を一時的な親天体とみなす
3. 親天体からの距離 `r` と親天体質量 `M` から円軌道速度を計算する
4. 速度方向は、親天体から対象天体への半径ベクトルに対して垂直方向とする
5. 公転方向は、ドラッグ終了時点の相対速度を優先して決める
6. その後は通常の N 体計算に戻す

円軌道速度:

```txt
v = sqrt(G * M / r)
```

この補正は、安定しやすい初期状態を作るための UX 補正であり、以降の運動を円軌道に固定するものではない。

### 5.5.4 削除

- 右クリックまたは Inspector の削除ボタンで削除候補を表示する
- 右クリックだけで即削除はしない
- 削除候補には、対象天体名と削除アクションを表示する
- Inspector の削除ボタンからも削除できる
- Inspector の削除ボタンを押した場合も、即削除ではなく確認 UI を挟む
- 太陽は削除不可
- 削除後、他天体の速度は自動補正しない
- ロケットなど一時的なオブジェクトは、必要に応じて確認なしで消せる例外として扱ってよい

---

## 5.6 新規惑星追加

空きスペースをドラッグして離すと、新しい惑星を追加できる。

- ドラッグ開始地点に新規惑星を生成する
- ドラッグ方向の反対向きに初速を与える
- ドラッグ距離が長いほど初速を大きくする
- ドラッグ中は予測軌道を表示する
- 新規惑星は他天体に重力影響を与える
- 質量、物理半径、表示半径は Tweaks の既定値を使う

---

## 5.7 入力ルール

操作競合を避けるため、入力ルールを以下に統一する。

| 操作 | 挙動 |
|---|---|
| 地球本体をドラッグ | 地球を移動 |
| 地球周辺の発射ハンドルをドラッグ | ロケット発射 |
| 地球以外の既存天体をドラッグ | 天体移動 |
| 空きスペースをドラッグ | 新規惑星追加 |
| Shift + ドラッグ | パン |
| 中ボタンドラッグ | パン |
| ホイール / ピンチ | ズーム |
| クリック | 選択 |
| 右クリック | 削除候補 |

### 5.7.1 地球ドラッグ操作の競合対策

地球はロケットの主な発射元である一方、惑星として位置移動の対象にもなるため、地球上のドラッグ操作は競合しやすい。

v1.0 では、誤操作を避けるため以下の方針を採用する。

- 地球本体のドラッグは、他の惑星と同様に地球の移動として扱う
- ロケット発射は、地球周辺に表示する発射ハンドルから行う
- 発射ハンドルは、地球選択時またはホバー時に表示する
- 発射ハンドルをドラッグした場合のみ、パチンコ式ロケット発射として扱う
- 発射ハンドルのドラッグ方向と距離から、発射方向と初速を決定する
- 発射ハンドルと地球本体の操作領域は、見た目とカーソルで明確に分ける

代替案として、Inspector に「配置モード / 発射モード」のトグルを置いてもよい。
ただし v1.0 では、初見ユーザーが迷いにくいように、発射ハンドル方式を推奨する。

現在の操作モードは、カーソル表示やガイドで明示する。

---

## 5.8 ズーム・パン

- ズーム範囲は `0.2x〜5x`
- ズーム中心はカーソル位置
- パンは Shift + ドラッグまたは中ボタンドラッグで行う
- 太陽系全体と地球周辺の両方を観察できるようにする

---

## 5.9 Follow Mode

選択中天体をカメラが追跡する。

- Inspector に Follow ボタンを配置する
- Follow 中はカメラ中心を対象天体に固定する
- Follow 中もズーム可能にする
- 対象天体が削除された場合は Follow を解除する
- Follow 中に手動パンした場合は、v1.0 では Follow を解除する

---

## 5.10 シミュレーション制御

以下を提供する。

- 再生
- 一時停止
- リセット
- 速度調整 `0.1x〜3x`
- 1 ステップ送り
  - 開発・デバッグ向け
  - UI 表示は任意

## 8.4 ロケットの物理扱い

ロケットは軽量なゲームオブジェクトとして扱う。

- 重力の影響を受ける
- 他天体へ重力影響を与えない
- ロケット同士の重力影響は計算しない
- 天体接触時は弾性衝突ではなく、着陸 / クラッシュ判定を優先する

推奨設定:

```ts
const rocket: Body = {
  type: "rocket",
  mass: 0,
  affectsGravity: false,
  affectedByGravity: true,
  landable: true,
  // ...
};
```

---

## 8.5 単位系

- 内部単位はピクセル系・無次元
- 実物理単位への変換は行わない
- G は無次元の「重力の強さ」として扱う
- 距離、速度、質量、半径は遊びやすさを優先して調整する

---

## 8.6 数値計算

### 積分方式

v1.0 では以下のいずれかを採用する。

1. Velocity Verlet
2. Leapfrog

明示オイラー法は初期検証用に限定し、v1.0 の通常実装には使わない。

### サブステップ

- 既定値は `4`
- G や質量が大きい場合に増やせるようにする
- 60fps を優先し、上限を設ける

### 衝突

| 対象 | 方針 |
|---|---|
| 惑星 vs 惑星 | 弾性衝突 |
| 惑星 vs 衛星 | 弾性衝突 |
| ロケット vs 惑星 | 着陸 / クラッシュ |
| ロケット vs 太陽 | 飲み込まれた判定 |
| ロケット vs ロケット | 原則無視 |
| UFO | 物理対象外 |

## 9. Rust/WASM I/F

## 9.1 方針

Rust/WASM は v1.0 必須ではないが、後から差し替えられる I/F を定義する。

- JS Solver と WASM Solver は同一 I/F を持つ
- JS と WASM 間の通信コストを抑える
- 将来的には `Float64Array` と `u32` ID を使う
- UI 表示用の `name` や `color` は JS 側に保持する

---

## 9.2 WASM エクスポート案

```rust
#[wasm_bindgen]
pub fn nbs_init(bodies_flat: &[f64], config: JsValue) -> u32;

#[wasm_bindgen]
pub fn nbs_step(handle: u32, dt: f64) -> *const f64;

#[wasm_bindgen]
pub fn nbs_body_count(handle: u32) -> u32;

#[wasm_bindgen]
pub fn nbs_add_body(
    handle: u32,
    mass: f64,
    physical_radius: f64,
    x: f64,
    y: f64,
    vx: f64,
    vy: f64,
    fixed: bool,
    affects_gravity: bool,
    affected_by_gravity: bool,
) -> u32;

#[wasm_bindgen]
pub fn nbs_remove_body(handle: u32, body_id: u32);

#[wasm_bindgen]
pub fn nbs_set_body(
    handle: u32,
    body_id: u32,
    x: f64,
    y: f64,
    vx: f64,
    vy: f64,
);

#[wasm_bindgen]
pub fn nbs_set_config(handle: u32, config: JsValue);

#[wasm_bindgen]
pub fn nbs_dispose(handle: u32);
```

## 9.3 JS 側ラッパー案

```ts
export interface Solver {
  init(bodies: Body[], config: SimConfig): void;
  step(dt: number): Body[];
  addBody(body: Omit<Body, "id">): number;
  removeBody(id: number): void;
  setBody(id: number, patch: Partial<Body>): void;
  setConfig(config: Partial<SimConfig>): void;
  dispose(): void;
}
```

`SolverJs` と `SolverWasm` はこの共通 I/F を実装する。

## 10.1 予測線・軌跡

- 予測線は、過去軌跡や惑星軌道と混同しない見た目にする
- 点線または短いセグメントで表現する
- ロケットの過去軌跡はフェード付きで表示する
- 惑星軌道ガイドは薄く表示する
- ドラッグ方向、発射方向、予測線の関係が一目でわかるようにする

## 11. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 通常 PC で 30 体・60fps を目標にする |
| 予測線 | ドラッグ中も体感で滑らかに更新される |
| 対応ブラウザ | 最新 Chrome / Safari / Firefox / Edge |
| モバイル | v1.0 対象外 |
| 言語 | v1.0 は日本語 UI |
| アクセシビリティ | 再生 / 停止 / リセットのキーボード操作、フォーカスリング |
| 保存 | Tweaks のローカル永続化のみ |
| 共有 | v1.0 対象外 |

---

## 11.1 パフォーマンス方針

- 入力応答と予測線の滑らかさを最優先する
- 予測計算が重い場合は精度を落としてよい
- Canvas 描画を基本とする
- DOM 要素の大量生成は避ける
- 軌跡・予測線には点数上限を設ける
- 30 体を超える天体は v1.0 の保証対象外

## 13. リスク

| ID | リスク | 対応 |
|---|---|---|
| R1 | 数値発散 | Velocity Verlet / Leapfrog、サブステップ、速度上限で緩和 |
| R2 | スケール差 | ズーム、パン、Follow Mode で対応 |
| R3 | 予測線の負荷 | 点数・秒数・更新頻度を制限 |
| R4 | 操作競合 | 入力ルールとガイド表示を明確化 |
| R5 | 衝突多発による負荷 | v1.0 では 30 体までを目標にし、空間分割は後回し |
| R6 | WASM 境界コスト | flat buffer と線形メモリ直接参照を検討 |
| R7 | Tweaks による破綻 | 範囲制限と上限値で制御 |

4. UFO の扱い
   - v1.0 では後回しを基本方針とする

   - v1.0 では Follow 解除を推奨する
